我需要你开发一个**本地运行的极简实时协作任务看板**，用于测试复杂架构的本地实现能力。

## 核心功能需求

### 1. 数据存储

- 使用本地安装的PostgreSQL作为主数据库，存储任务数据（任务ID、标题、描述、状态、创建时间、更新时间）

- 使用本地安装的Redis作为辅助存储，用于加速读取和实现实时功能

### 2. 后端服务

- 提供HTTP API供前端调用，实现任务的增删改查

- 实现WebSocket或类似的长连接机制：当一个用户修改任务时，其他打开页面的用户能立即看到变化（无需刷新页面）

- 实现简单的缓存策略：频繁查询的任务列表可以缓存在Redis中，数据变更时自动使缓存失效

### 3. 前端界面

- 单页应用，打开浏览器即可使用

- 界面包含：

- 输入区域：添加新任务（标题和描述）

- 任务列表：显示所有任务，支持修改状态（如下拉选择待办/已完成）和删除

- 实时状态指示器：显示当前连接状态和最近的操作通知（如"刚刚有人更新了任务"）

- 界面风格简洁美观，响应式设计（适配手机和电脑）

### 4. 本地开发体验

- 提供清晰的环境准备说明（如何安装和启动PostgreSQL和Redis）

- 提供一键启动方式（或最简启动步骤）

- 包含数据库初始化脚本（自动创建表结构）

## 约束条件（不要违反）

1. **极简原则**：只实现任务管理这一个核心功能，不要用户系统、不要权限管理、不要登录注册

2. **本地优先**：所有服务都运行在本地电脑，不依赖云服务或第三方SaaS

3. **单机并发安全**：考虑多个浏览器标签同时操作时的数据一致性（使用最简单的方式处理即可）

4. **数据持久化**：重启程序后，已创建的任务不能丢失

## 交付物要求

1. **完整源代码**：包括后端代码、前端代码、配置文件

2. **环境说明文档**：列出需要本地安装的软件（PostgreSQL和Redis）及版本要求，提供各操作系统（Windows/macOS/Linux）的安装命令或下载链接

3. **启动指南**：从克隆代码到看到界面的详细步骤，包括数据库初始化命令

4. **架构说明**：简要说明你选择的技术栈及原因（比如你选择了什么后端框架、前端框架、为什么选择它们）

## 测试验收场景

我会按以下流程验证你的实现：

1. 启动PostgreSQL和Redis本地服务

2. 运行你的程序，打开浏览器访问界面

3. 添加3个任务，确认显示正常

4. 打开第二个浏览器标签（或不同浏览器），确认能看到这3个任务
